# 行为准则

本项目欢迎围绕撮合内核、节点服务、复制、高可用、运维和 SDK 的技术贡献。参与讨论、提交 issue、review 或 Pull Request 时，请保持专业、清晰和尊重事实。

## 期望行为

- 围绕具体代码、设计、数据和可复现结果讨论问题
- 尊重不同经验背景的贡献者
- 在指出问题时给出可操作的复现、证据或改进建议
- 对性能、安全和可靠性结论标明测试环境和约束
- 避免把无关业务、营销内容或人身评价带入项目讨论

## 不接受的行为

- 人身攻击、骚扰、歧视或威胁
- 泄露他人隐私、凭证、密钥或未公开漏洞细节
- 在公开 issue 中发布可被直接利用的安全漏洞细节
- 反复提交无关内容、广告或低信号噪音

## 处理方式

维护者可以编辑、隐藏或删除不符合本准则的内容，也可以限制相关账号继续参与项目讨论。安全漏洞请按 [SECURITY.md](SECURITY.md) 的方式报告。
