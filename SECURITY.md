# 安全策略

## 支持版本

当前仓库只对最新 `master` 分支状态提供安全修复支持。

| 版本 | 是否支持 |
| --- | --- |
| 最新 `master` | 是 |
| 更早提交或分叉版本 | 否 |

## 漏洞报告方式

发现疑似安全漏洞时，不要直接提交公开 GitHub Issue。

请使用以下方式之一：

1. 仓库启用时，使用 GitHub 私有漏洞报告
2. 直接联系仓库维护者

报告中请至少包含：

- 受影响模块
- 精确版本或 commit
- 复现步骤
- 影响评估
- 是否影响机密性、完整性、可用性或运行隔离

## 响应预期

- 首次确认目标：5 个工作日内
- 分诊结果：确认、拒绝或要求补充细节
- 修复策略：优先在 `master` 上修复；若发布行为发生变化，同步更新 `README.md` 或相关运维文档

## 关注范围

本项目是撮合引擎与撮合节点服务，安全敏感区域包括：

- gRPC TLS 与 mTLS 配置
- ZooKeeper lease / fencing 处理
- 服务发现元数据
- WAL、快照与重放完整性
- readiness、failover 与复制控制路径
